DocIntel — invoice extraction that improves itself

BIO sequence-labeling from a rule baseline up to a BiLSTM+CNN, wrapped in the production ML machinery that's usually skipped: confidence calibration, active learning, a champion/challenger gate, and drift detection. Every number below is measured, including the ones where the fancy model loses.

1 · The model ladder

Field macro-F1 on hard synthetic invoices, GroupKFold by document. Colour = tier.

Random Forest on engineered features (0.86) beats every neural model here. At a few-hundred-document scale the sequence models can't pay for themselves — and multi-seed error bars show the RNN/LSTM/BiLSTM gaps are within noise.

2 · The self-improving loop, on real data

Feed 150 real invoices → retrain through the champion/challenger gate on a frozen real holdout.

Real corrections in, a measurably better and better-calibrated model out. Promoted only after a paired-bootstrap significance test on the held-out documents.

3 · Active learning — label efficiency

XGBoost, seed 20 docs, batch 10, averaged over 3 seeds. Field-F1 vs. number of labels.

Least-confidence sampling reaches random's 100-label plateau with ~40 labels — ~60% less labeling. Margin+diversity underperforms here: with only 8 layout families, forced cluster coverage wastes budget (it should pay off with real vendor variety).

4 · Serving — ONNX INT8 quantization

BiLSTM, single CPU thread, per-document latency. Lower is better.

INT8: 3.9× faster at p95 (4.3 ms → 1.1 ms), 3.9× smaller (1.34 MB → 0.35 MB), −0.004 F1.
